# Commit Convention
[Feat] : 새로운 기능 추가

[DB_feat] : 데이터베이스 기능 추가

[Fix] : 버그 수정

[DB_fix] : 데이터베이스 수정

[Docs] : 문서 수정

[Test] : 테스트 코드

[Remove] : 기능 혹은 내용 삭제

[Design] : 스타일 변경

# Pull requests Convention
1. PR시 코드 확인 후 반드시 Comment 작성.

2. PR시 제목은 Commit한 내용과 동일하게 작성.

3. PR 내용은 코드 전반적인 내용과 설명 작성.
  
   변경 내용이 화면 상으로 구현 시 사진 및 동영상 첨부
